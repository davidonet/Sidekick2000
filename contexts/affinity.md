# Formation Affinity (Serif) — Contexte

Ce document fournit le contexte pour les synthèses de sessions de formation à la suite Affinity dispensées par AKER.

---

## Cadre de la formation

| Champ                | Valeur                                                         |
| -------------------- | -------------------------------------------------------------- |
| **Suite logicielle** | Affinity (Serif) — suite de PAO / création graphique           |
| **Formateur**        | Adrien Revel                                                   |
| **Public**           | Graphistes indépendants, Architectes, Chargés de communication |
| **Format**           | Sessions de formation pratique                                 |

---

## Affinity — Les applications

| Application            | Rôle                                                                  |
| ---------------------- | --------------------------------------------------------------------- |
| **Affinity Designer**  | Illustration vectorielle et design graphique (équivalent Illustrator) |
| **Affinity Photo**     | Retouche photo et édition d'images (équivalent Photoshop)             |
| **Affinity Publisher** | Mise en page et publication (équivalent InDesign)                     |

---

## Concepts et vocabulaire clés

| Terme                         | Définition                                                                                  |
| ----------------------------- | ------------------------------------------------------------------------------------------- |
| **Persona**                   | Mode de travail dans une app Affinity (ex. Designer Persona, Pixel Persona, Export Persona) |
| **Artboard**                  | Plan de travail indépendant dans un même document                                           |
| **Layer / Calque**            | Couche d'empilement des objets dans la composition                                          |
| **Master Page / Gabarit**     | Page modèle réutilisable (Publisher)                                                        |
| **Spread / Planche**          | Double page en vis-à-vis (Publisher)                                                        |
| **Frame Text / Bloc texte**   | Zone de texte chaînable entre pages (Publisher)                                             |
| **Linked File / Fichier lié** | Ressource externe placée dans le document (image, .afdesign, .afphoto)                      |
| **StudioLink**                | Fonctionnalité permettant de passer entre Designer/Photo/Publisher sans quitter l'app       |
| **Snapping / Magnétisme**     | Accrochage automatique aux guides, grilles et objets                                        |
| **Constraints / Contraintes** | Règles de positionnement responsive d'un objet par rapport à son parent                     |
| **Symbols / Symboles**        | Objets réutilisables et synchronisés entre instances                                        |
| **Adjustment Layer**          | Calque de réglage non destructif (niveaux, courbes, TSL…)                                   |
| **Macro**                     | Séquence d'opérations enregistrée et rejouable                                              |
| **Preflight**                 | Vérification avant export/impression (Publisher)                                            |
| **PDF/X**                     | Format d'export normé pour l'impression professionnelle                                     |
| **Bleed / Fond perdu**        | Marge d'impression au-delà du bord de coupe                                                 |
| **Slug / Zone technique**     | Zone hors impression pour annotations techniques                                            |
| **CMJN / CMYK**               | Espace colorimétrique pour l'impression                                                     |
| **RVB / RGB**                 | Espace colorimétrique pour l'écran                                                          |
| **DPI / PPP**                 | Résolution d'image (points par pouce)                                                       |

---

## Consignes de synthèse

- Le formateur est **Adrien Revel** — identifier ses interventions comme telles (il guide, explique, montre).
- Les autres participants sont des **apprenants** — noter leurs questions et difficultés rencontrées.
- Structurer les notes par **thème ou fonctionnalité** abordée plutôt que strictement par ordre chronologique.
- Préciser quelle **application Affinity** est concernée (Designer, Photo ou Publisher) quand c'est identifiable.
- Inclure une section **Exercices / Manipulations** si des cas pratiques ont été réalisés.
- Inclure une section **Questions / Réponses** pour les échanges notables.
- Inclure une section **Raccourcis et astuces** mentionnés par le formateur.
